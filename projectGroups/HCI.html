WebLingo is a project that aims to help people who are either learning a new language, or are trying to improve their English. The project is still in development, but it aims to provide real world examples of languages being used on the web.


First Iteration

In this first iteration of designing Weblingo, we individually took into account the feedback we recieved from our interviews held. The key aspects are:

  • Translation: The translation feature allows users to translate any text they find on the website. This feature would be first thing someone who is using a language chrome extention would expect.
  • Pronouciation: Being able to hear the word you are trying to learn would be an extra step of study for anyone learning new vocabulary.
  • Quizzes: One of the main parts of the learning aspect of the extention is the quiz function which both of the interviews I conducted agreed would be benificial to have.


Second Iteration


In the second iteration of designing Weblingo, we took into account all of the feedback we recieved from each of our interviews that we held. Having have taken a fictional inquiry aspect towards collecting data, and having provided images to draw on, we used these as inspiration alongside with what was said about what someone would be looking for in order to decide on our own version of Weblingo. Quotes such as "stay up to date in terms of the new words that are formed on the internet so that if I highlight something that's like probably a new trend. It can still understand it..." help us understand what the user would be looking for in a language learning extention.
Looking at everyones transcripts, we analyzed the feedback using an affinity diagram, and create codes that would later shape our requirements. After looking at what everyone came up with, we reached a consensus and created 3 main requirements for the project.

Requirements

    Translation: Changes words from one language to another, as Users want to get the meaning of the words directly.
  • Interaction Flow: The User can click on the translation button and then hover over different text on the webpage they are on to see the traslation of the word.

    • Quizzes: The plugin will quiz users with either multiple-choice or free response questions, as Users want to actively participate in the learning process and make sure they actually know the words they are trying to learn.
  • Interaction Flow: The User can click on the quiz section from the main page and then proceed to take a quiz with their score being accurately recorded.

    • Remembering: History of translated words: The plugin will have a history bar where it stores words that the user has previously translated, as Users want to remember words/phrases learned as language learners.
  • Interaction Flow: The User can click on the history section from the main page and then be taken to a page which would ideally contain all of the words they have seen throughout their time studying.

The main focus of the second iteration was to create a more user friendly experience for the user. I wanted to make sure that the user could easily see what the expected functionality of the trasnaltion, the quizzes, and the history sections would be, and therefore focused most of my time on creating the functionality of those aspects.
